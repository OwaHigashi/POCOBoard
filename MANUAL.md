# POCOBoard — 使い方マニュアル

配信中に「操作ウィンドウ」と「全画面表示ウィンドウ」の **2画面運用** で使う、
Windows向けの演出アプリです。同じLAN内のブラウザから複数人が同時にリクエスト
（BOMB / CHEER / HEARTS / STARS / SNOW / TALK / 横スクロール文字）を飛ばせます。
TALK は同時発話でも自動でミックスされます。

---

## 1. 動作要件

- Windows 10 / 11（64-bit）
- ディスプレイ 1枚でも動きますが、**2枚以上**で真価を発揮します
- スピーカー（内蔵でも外付けでもOK）
- ネットワーク：Wi-Fi または LAN。配信用PCとスマホ/他PCが**同じネットワーク**にいること
- 送信元ブラウザ（スマホ/PC）は最近のもの（Edge / Chrome / Firefox / Safari 16+）

exe版を使うだけなら Python のインストールは **不要** です。

---

## 2. インストール方法（他のパソコンへ配布する場合）

### A. 配布用の exe 版（推奨・一番かんたん）

1. 作者から `POCOBoard.zip` を受け取る（または自分でビルド → セクション **7** 参照）
2. ZIP を好きな場所に展開する。例：`C:\POCOBoard\`
3. 展開後のフォルダ内にある `POCOBoard.exe` を**ダブルクリック**
4. 初回起動時に Windows Defender SmartScreen が出たら「詳細情報」→「実行」を選択
5. （任意）`config.example.ini` を同じフォルダに `config.ini` としてコピーし、好みを編集

> フォルダ中身のイメージ
> ```
> POCOBoard\
>   POCOBoard.exe            ← これをダブルクリック
>   config.example.ini       ← 必要ならコピーして config.ini に
>   _internal\               ← Qt DLL など（触らないでOK）
> ```

### B. ソース版（開発・改造したい人向け）

1. Python 3.10 以上をインストール（[公式](https://www.python.org/downloads/)）
   - インストーラで **「Add python.exe to PATH」** にチェック
2. `POCOBoard` フォルダを丸ごとコピー
3. `install-deps.bat` をダブルクリック（`PySide6` が入ります）
4. `run.bat` をダブルクリックで起動

---

## 3. 初回の起動

`POCOBoard.exe`（または `run.bat`）を実行すると、**2つのウィンドウ** が開きます。

| ウィンドウ | 役割 | サイズ |
|---|---|---|
| **Control（操作）** | オペレータ（配信者自身）が使う固定サイズのパネル | 820 × 1020 |
| **Display（表示）** | 視聴者に見せる大画面（FX / 横スクロール / 動画） | 全画面 or 最大化可 |

モニタが2枚あるとき、Display ウィンドウは **自動的に Control と反対側の画面** に
配置されます（デフォルト設定 `display_screen = -1`）。モニタが1枚しかない場合は、
同じ画面に両方出ます（Display は手動で移動/最小化してください）。

### 起動後、最初にすること

1. **Control ウィンドウ**を確認する
2. 上のほうに表示されているリモートURL（例：`http://192.168.1.23:8080/`）を控える
3. スマホ/他PCのブラウザで、そのURLを開く

---

## 3.5. 表示ウィンドウのレイヤー構造

Display は下から順に重なる4層で描画されます:

```
  ┌────────────────────────── Display ──────────────────────────┐
  │ 4) 横スクロール（常に最前面）                                   │
  │   ─────────────────────────────────────────────────           │
  │ 3) FX (BOMB / CHEER / 等 — 一時的なエフェクト)                  │
  │   ─────────────────────────────────────────────────           │
  │ 2) 背景: 画像 or 動画（表示削除まで残る／動画は無限ループ）       │
  │   ─────────────────────────────────────────────────           │
  │ 1) ベース: 黒 or タイトル画面                                   │
  └─────────────────────────────────────────────────────────────┘
```

**各層の振る舞い**
- **ベース** — 起動直後は「POCOBOARD」タイトル。何かリクエストが来たら黒へ。
  **5分間リクエストが無い** と自動でタイトルへフェードバック。
- **背景** — 画像アップロードで画像背景、動画アップロードで動画（無限ループ）、
  音声アップロードで音声（無限ループ、画面には何も出ない）。
  **オペレータが ⛔ 表示削除 ボタンを押すまで消えません。**
- **FX** — BOMB/CHEER 等、数秒で終わる演出。背景動画の上では 75% 透過で重ねます。
- **横スクロール** — 常に最前面。FX中でも読めます。

つまり、写真/動画は**「背景に張り付いて消えない」**、文字スクロール演出は
**「いつでも上に重ねて流せる」**、という配信向けの落ち着いた振る舞いです。

## 4. 制御ウィンドウ（オペレータ側）の使い方

操作パネルは **常時表示のヘッダ** と **下部タブ** の二層構造です（820×880）。

**常時表示（パネル上部）**
- タイトル + `ACCEPT / REJECT` トグル
- ステータス (リモートURL、メッセージ流数、接続中クライアント数)
- エフェクト 6 ボタン (BOMB / CHEER / HEARTS / STARS / SNOW / MARQUEE STOP)
- **⛔ 表示削除** ボタン — 背景画像・動画・音声をまとめて停止
- 音量スライダ

**タブ**
- 📥 **キュー** — アップロードされた画像/動画/音声の待ち行列管理
- 📢 **横スクロール** — メッセージ作成、タグボタン、速度、流す/停止
- 🖥 **表示** — 出力スクリーン選択、F11、ローカル画像/動画/音声を開く
- 👥 **ユーザー** — 接続中クライアント一覧（スクロール）、各人の許可/拒否トグル、全員許可/全員拒否
- 📜 **ログ** — リクエストログ（拡大表示）、ログ消去

### 4.1. キュータブ — メディア待ち行列

ブラウザから送られてきた画像/動画/音声は **自動再生されず**、いったん
キュー（待ち行列）に入ります。オペレータが順に捌いていきます。

```
┌─ 📥 キュー ─────────────────────────────────┐
│ [⛔ 停止 (背景・動画・音声を止める)]  [全削除] │
│ ┌─────────────────────────────────────────┐ │
│ │ 再生中: 🎬 intro.mp4  (Alice)           │ │
│ └─────────────────────────────────────────┘ │
│                                              │
│ [📷 IMG] party.jpg  2.4MB  — Alice  [▶ 再生][🗑] │
│ [🎬 VID] welcome.mp4 12MB  — Bob    [▶ 再生][🗑] │
│ [🎵 MP3] bgm.mp3     3.1MB — Carol  [▶ 再生][🗑] │
└──────────────────────────────────────────────┘
```

- **`[▶ 再生]`** — そのアイテムを今すぐ出し、キューから外す
- **`[🗑 削除]`** — 再生せずに捨てる（ファイルもディスクから削除）
- **`[⛔ 停止]`** — 現在再生中の背景画像・動画・音声を止める（キューは残る）
- **`[キュー全削除]`** — 待機中のすべてを一括で捨てる
- 画像/動画は同じ「視覚スロット」を使うので片方しか再生できません。新しい画像/動画を再生すると前の視覚は置き換わります。
- 音声は別スロットなので、背景画像＋BGMの同時再生も可能です。

```
┌────────────── POCOBoard — Control ──────────────┐
│ POCOBoard                         [ ACCEPT ]    │
│ ─────── ステータス ───────                        │
│ リモートURL: http://192.168.1.23:8080/           │
│ marquee 0/9 レーン使用中 / 接続中クライアント: 2    │
│ ─────── エフェクト ───────                         │
│ [ BOMB ]  [ CHEER ]  [ HEARTS ]                 │
│ [ STARS ] [ SNOW  ]  [ MARQUEE STOP ]           │
│ ─────── 音量 ───────────────────                  │
│ ━━●━━━━━━━━━━━━━━━  80 / 100                     │
│ ─────── 横スクロール ──────                        │
│ [本文入力欄]                                      │
│ [赤][黄][緑] ... [小][中][大] [下線][強調]          │
│ [x1] [x2] ... [x5]  [流す] [停止]                │
│ ─────── 表示ウィンドウ ────                        │
│ 出力スクリーン: [Screen 1]  [フルスクリーン(F11)]  │
│ [動画ファイルを再生...]     [動画停止]             │
│ ─────── リクエストログ ────                        │
│ 14:32:18  Alice (#51c66102)    BOMB             │
│ 14:32:22  #2e6807a4            HEARTS           │
│ 14:32:25  Bob (#2e6807a4)      MARQUEE x2  ... │
│ ...                                              │
│ [ログ消去]                                        │
└─────────────────────────────────────────────────┘
```

- **ACCEPT / REJECT ボタン（右上）**
  緑 `ACCEPT`：ブラウザからのリクエストを受け付ける。
  赤 `REJECT`：すべて拒否（配信の流れを止めたくないとき用）。
  ローカル操作（ここのボタン押下）は `REJECT` 中でも動きます。

- **エフェクトボタン**
  押すと Display ウィンドウにエフェクト＋音が出ます。debounce（連打防止）
  が300msかかるので、連打しても暴発しません。

- **音量スライダ**
  0〜100。FX/TALK 両方に適用されます。

- **横スクロール**
  本文欄に文字を入れ、必要なら `[赤]` `[大]` などのタグボタンで装飾し、
  `[x1]～[x5]`で速度を選び、`[流す]` で Display に流します。
  **同時メッセージ数の上限はありません** — 画面が埋まると自動で重なります（ニコニコ風）。
  `[上]` (ue) / `[下]` (shita) ボタンで、本文冒頭にプレフィックスを付けると
  画面上部/下部に **3秒間ピン留め** 表示（Niconicoの `<ue>` / `<shita>` コマンド相当）。

- **動画ファイルを再生**
  ローカルの `.mp4 / .mkv / .mov / .avi / .webm / .m4v / .wmv` を Display
  に全画面再生します。`[動画停止]` または再生完了で元の画面に戻ります。

- **フルスクリーン（F11）**
  Display ウィンドウの全画面切替。F11 キーでも切り替え可。ESCで抜けます。

- **リクエストログ**
  だれが（ブラウザ固有のCookieIDまたは表示名）何を送っているかを
  色分けで流れるように表示。TALK はクライアント毎に2.5秒おきに1行に
  まとめられるので、流れ続けてもログが埋まりません。

## 4.5. ユーザー管理（ブロック/許可）

制御ウィンドウの一番下に「ユーザー管理」セクションがあります。

```
┌─ ユーザー管理 ─────────────────────────────────┐
│ [全員を許可] [全員を拒否]         3 人  [更新] │
│ ● Alice  (#51c66102)             [✓ 許可中]   │
│ ● Bob    (#2e6807a4)             [🚫 拒否中]  │
│ ● #a3c9...                        [✓ 許可中]   │
└────────────────────────────────────────────────┘
```

- ブラウザが初めてアクセスすると自動的にリストに追加され、**デフォルトで許可** 状態
- 各行の「✓ 許可中 / 🚫 拒否中」ボタンで個別に切り替え
- `[全員を許可]` / `[全員を拒否]` ボタンで一括操作
- `[更新]` で最新状態に再描画（通常は自動更新）
- LED の緑 = 30 秒以内にアクセスあり、グレー = アイドル

拒否されたクライアントのリクエストは **403 blocked** で返され、Display
に何も反映されません。ブラウザ側ではオペレータに拒否されている旨の
トーストが出ます。

## 5. ブラウザ（リモート側）の使い方

Control ウィンドウに表示された URL を、同じLANのスマホやPCのブラウザで開きます。

```
┌─ POCOBoard ─────────────────────────────────────┐
│ [表示名 入力欄]  #abcd1234    受付:ON  音量:80    │
│                                                  │
│ [ BOMB ]  [ CHEER ]  [ 🎙 TALK ]                  │
│ [HEARTS]  [ STARS ]  [ SNOW  ]                  │
│                                                  │
│ 📢 横スクロール                  0 / 200          │
│ [本文]                                           │
│ [赤][黄]...[小][大][下線] [x1][x2]...[流す][停止] │
└──────────────────────────────────────────────────┘
```

### 表示名
左上の入力欄に名前を入れておくと、オペレータのログに
`Alice (#51c66102)` のように表示されます（空欄なら `#51c66102` 形式）。
**Cookie に保存されるので、次回同じブラウザで開いたとき自動復元** されます。

### TALK（マイク）
`🎙 TALK` をタップするとマイク許可を求められます。許可すると録音＆送信が始まり、
もう一度タップで停止。
- **HTTPS または `localhost` でアクセス**する必要があります（ブラウザの制約）
- LAN で `http://…` のまま使いたい場合は、PC側で localhost（`http://127.0.0.1:8080/`）を開くか、逆プロキシでHTTPS化してください

### 複数人の同時発話
複数人が同時に TALK しても **サンプル単位で加算ミックス** され、きれいに重なって
聞こえます（Int16 レンジ内で飽和クリップ）。

### 📷 写真 / 🎬 動画 / 🎵 音声 を投稿する

ボタン下の緑パネルから、ローカルのメディアファイルを選んで Display に出せます。

| ボタン | 挙動 | 上限 |
|---|---|---|
| 📷 写真を送る | Display の**背景**として表示（レターボックス、送信者名キャプション付き） | 25 MB |
| 🎬 動画を送る | Display の**背景**として無限ループ再生 | 200 MB |
| 🎵 音声ファイル | スピーカーから無限ループ再生（TALK とミックス） | 50 MB |

**重要: 画像 / 動画 / 音声は「背景」として表示され続けます** — 次の画像/動画が
来るか、オペレータが **「⛔ 表示削除」** ボタンを押すまで消えません。
BOMB などのエフェクトや横スクロールメッセージは、この背景の上に重ねて表示されます。

- 対応形式: 画像 `jpg/png/webp/gif/bmp` / 動画 `mp4/mkv/mov/avi/webm/m4v/wmv` / 音声 `mp3/wav/m4a/aac/ogg/flac`
- アップロード中はプログレスが % 表示されます
- 受付OFF（ACCEPT→REJECT）や、個別に拒否されているユーザーからはエラーが返ります
- アップロードされたファイルは `POCOBoard.exe` 隣の `cache\uploads\` に保存、**50 件**を超えると古いものから自動削除

## 6. 設定ファイル `config.ini`

`POCOBoard.exe` と同じフォルダに置けば起動時に読み込まれます。
雛形は `config.example.ini` としてすぐ横にあるので、コピーして書き換えてください。

```ini
# ---- Network ----
http_host       = 0.0.0.0        # すべてのNICでLISTEN（変更不要）
http_port       = 8080           # URL の末尾ポート番号

# ---- Audio / behaviour ----
startup_volume  = 80             # 0..100
accept_on_boot  = true           # 起動時に受付ONか
debounce_ms     = 300            # FX連打防止

# ---- Display window ----
display_screen  = -1             # -1=反対側, 0..n=指定スクリーン
display_fullscreen_on_boot = true
display_width   = 1600           # 非全画面時の初期サイズ
display_height  = 900

# ---- Control window ----
control_screen  = -1             # -1=プライマリ

# ---- Marquee ----
marquee_size    = 64             # 横スクロール文字サイズ(px)
marquee_ref_lanes      = 5
marquee_wait_per_empty = 3
```

コマンドライン引数でも上書き可能:

```bat
POCOBoard.exe --port 9000 --no-fullscreen --display-screen 1
```

## 8. トラブルシューティング

| 症状 | 対処 |
|---|---|
| `ポート 8080 をバインドできませんでした` | 他のアプリが使用中。`config.ini` の `http_port` を別の値（例 `8888`）に変更 |
| ブラウザからURLに繋がらない | 同じLANにいるか確認／Windows Defender ファイアウォールで Python / POCOBoard.exe を許可 |
| ブラウザで TALK が使えない | `http://localhost:8080/` で開くか、HTTPS プロキシ経由にする（ブラウザの仕様） |
| 動画が再生されない（黒いまま） | Windows の codec が足りない可能性。K-Lite Codec Pack などをインストール、または別形式（`.mp4` H.264）に変換して試す |
| Display ウィンドウが思った画面に出ない | Control ウィンドウの「出力スクリーン」を切り替える、または `config.ini` の `display_screen` を `0` / `1` / `2` と明示指定 |
| SmartScreen が実行を阻止 | 「詳細情報」→「実行」。社内配布なら管理者で署名することを検討 |
| FX や音が出ない | Control ウィンドウの音量が0になっていないか／Windows の既定再生デバイスを確認 |
| ウィンドウが残り方がおかしい | F11 で全画面トグル、または Alt+F4 で閉じてから再起動 |

## 8.5. 横スクロール用のタグ一覧（ニコニコ動画コマンド風）

**位置（先頭に置く・閉じタグ不要）**
- `<ue>` / `<top>` — 上部中央に3秒間固定
- `<shita>` / `<bottom>` — 下部中央に3秒間固定
- `<naka>` / `<middle>` — スクロール（デフォルト）

**色（`</>` または `</color>` で閉じる）**
- 短縮形: `<r> <g> <b> <y> <c> <m> <w> <o>`
- 長い名前: `<red> <green> <blue> <yellow> <cyan> <purple> <white> <orange> <pink>`

**サイズ（それぞれの終了タグで閉じる）**
- `<small>` / `<s1>` — 小
- `<normal>` / `<s2>` — 中（デフォルト）
- `<big>` / `<s3>` — 大

**装飾**
- `<u>...</u>` — 下線
- `<hl>...</hl>` または `<mark>...</mark>` — ハイライト（黄背景）

**例**

```
<ue><big><y>19時開始！</y></big>
<r>重要</> <big>サプライズ準備</big> <u>集合してください</u>
<shita><pink>ありがとう〜</pink>
```

## 10. HTTP API（自作ツールから叩きたい人向け）

すべてのエンドポイントは `http://<IP>:<port>/` からの相対パス。
`poco_client` Cookie で発信元を識別、`X-Poco-Name` ヘッダで表示名を指定できます。

| Method | Path | Body | 説明 |
|---|---|---|---|
| GET | `/` | — | リモートUI (HTML) |
| GET | `/status` | — | JSON: `{accept, volume, clients, marquee, me}` |
| POST | `/bomb` | — | BOMB発動 |
| POST | `/clap` | — | CHEER発動 |
| POST | `/hearts` | — | HEARTS発動 |
| POST | `/stars` | — | STARS発動 |
| POST | `/snow` | — | SNOW発動 |
| POST | `/talk?sr=16000` | Int16LE mono PCM（16–48kHz） | スピーカーへ送出（ミックス対応） |
| POST | `/marquee?speed=1..5` | UTF-8 テキスト（タグ可） | 横スクロール追加（上限なし・重なり許可） |
| POST | `/marquee/stop` | — | 全横スクロールを停止 |
| POST | `/name` | `{"name": "表示名"}` | 表示名を Cookie に保存 |
| POST | `/upload?type={image\|video\|audio}&filename=...` | 生バイナリ (Content-Type に MIME) | メディアを投稿 |

拒否時は次のJSONを返します（ステータスコードに意味があります）:

| Code | reason | 意味 |
|---|---|---|
| 503 | `disabled` | グローバル REJECT 状態 |
| 403 | `blocked` | 当該ユーザーがブロック済み |
| 429 | `busy` | FX debounce 内 |
| 400 | `empty` / `not_utf8` / `bad_type` | 不正リクエスト |
| 413 | `too_large_or_empty` | アップロードサイズ超過 |

例:

```bash
curl -X POST http://192.168.1.23:8080/bomb
curl -X POST -H 'Content-Type: text/plain; charset=utf-8' \
     --data-raw '<r>お知らせ</> <big>開始！</big>' \
     'http://192.168.1.23:8080/marquee?speed=2'
```

## 11. 他のパソコンへ持って行く一番速い手順まとめ

1. 開発元PCで `build.bat` 実行
2. `dist\POCOBoard\` を右クリック → 「送る」→「圧縮(ZIP)フォルダ」
3. 生成された `POCOBoard.zip` を USB / Slack / メール等で配布
4. 受け取った側は ZIP を展開 → `POCOBoard.exe` をダブルクリック → 完了

Python も Qt も **相手のPCには何も入れなくてOK** です。
